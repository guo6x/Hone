Windows 系统代理透传

自动读取注册表系统代理，注入到所有子进程。你配了一次系统代理，Gateway 和 CLI 都自动走。

  • 读取 HKCU\...\Internet Settings 代理设置
  • 自动注入 HTTP_PROXY / HTTPS_PROXY / ALL_PROXY
  • NO_PROXY 自动包含 localhost / 127.0.0.1
  • 用户已设环境变量则不覆盖
# 注册表读取
ProxyEnable: 0x1
ProxyServer: 127.0.0.1:7890
# 注入子进程
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
NO_PROXY=localhost,127.0.0.1
✓ 所有子进程自动走代理

安全

密钥永不裸奔

所有敏感信息走 OS 凭据存储，绝不写入明文 JSON。配对码一次性，Relay 只存派生证明。

🔐

OS 凭据存储

API Key、Relay Token、配对码全部通过系统 keyring 保存（Windows Credential Manager / macOS Keychain）。gateway-config.json 里看不到任何密钥。

🔑

一次性配对码

6 位数字配对码，配对成功后立即清空。Relay 只保存 SHA-256 派生证明，无法反推原码。设备凭据独立持久化。

🚫

设备撤销

桌面端可随时踢掉已配对设备，被撤销设备立即断连并清除凭据。重新连接必须重新扫码配对。
